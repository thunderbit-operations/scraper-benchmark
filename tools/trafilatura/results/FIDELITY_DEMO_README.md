# trafilatura — real-article extraction fidelity demo (results)

Non-timed capability demonstration, **not** a scored benchmark. No gold standard,
so no F1 / precision / recall is computed. Generated by
`../tests/run_trafilatura_fidelity_demo.py` (offline; reads saved fixtures, no
network at runtime). trafilatura 2.1.0.

## Reproduce
```bash
cd tools/trafilatura
python3 -m venv .venv && .venv/bin/pip install trafilatura
.venv/bin/python tests/run_trafilatura_fidelity_demo.py
```
The script auto-detects the layout (`fixtures/real` + `results/` here). It reads
the two saved HTML fixtures and writes the outputs below.

## Files
- `trafilatura-fidelity-summary.json` — machine summary (per-fixture byte counts,
  boilerplate drop-check, metadata field hits, SHA-256 cross-check).
- `fidelity_<name>.txt` / `.md` / `.json` — extracted body / markdown / metadata per fixture.

## What the run showed (from the summary JSON, not hand-typed)
- **Wikipedia**: 230,049 raw HTML bytes → 26,673 extracted body bytes (~0.116 body/raw).
  All 4 checked chrome markers dropped; title/date/hostname captured; author/sitename null.
- **Wikinews**: 79,716 raw HTML bytes → 2,200 extracted body bytes (~0.028 body/raw).
  All 5 checked chrome markers dropped; title/date/hostname captured; author/sitename null.

Boundary noted honestly: the byte-ratio measures how much markup+chrome was stripped,
**not** an accuracy score. Extracted body includes article-embedded content such as
Wikipedia's citation list and a maintenance banner (article content, not site chrome).
`author`/`sitename` came back null on both MediaWiki pages — reported as misses, not hidden.
