# MarkItDown fixtures — how to reproduce

Small fixtures are committed as examples. Four large fixtures are **not**
committed (size / provenance) and must be regenerated or re-downloaded before
running the scale benchmark (`tests/scale_bench.py`). Everything else the
runners need is already here.

## Committed (kept as examples)
- Shared web HTML: `books_toscrape.html`, `quotes_toscrape.html`,
  `scrapethissite_forms.html`, `wikipedia_web_scraping.html` — saved copies of
  the four shared review fixtures (originals via `curl`, see below).
- `tables/` — the 13 complex-table cases (`t01`..`t13`) + `manifest.json`
  (pre-registered ground truth). Regenerate with `python tests/make_fixtures.py`.
- `complex_tables.html` — the 13 cases combined on one page (also regenerated).
- `docs/` small documents: `bitcoin.pdf` (9pp), `sample.pdf`,
  `scanned_financial.pdf` (image-only, **0 text layer**, verified),
  `mid_test.docx` / `mid_test.xlsx` / `mid_test.pptx` / `mid_equations.docx` /
  `mid_test_outlook_msg.msg` (from markitdown's own **MIT** test suite),
  `wide_sheet.xlsx` (200×64, synthetic — regenerated by `make_fixtures.py`).

## NOT committed — regenerate or download

### Synthetic, regenerate deterministically (fixed seed)
```bash
python tests/make_fixtures.py
# writes docs/large_sheet.xlsx  (1 sheet x 50,000 rows x 8 cols)
#        docs/wide_sheet.xlsx   (200 rows x 64 cols)  [small; already committed]
#        tables/*.html + tables/manifest.json + complex_tables.html
```

### Public documents, re-download
```bash
cd docs

# arXiv "Attention Is All You Need" — 15pp text-layer PDF with real tables/refs
curl -sL "https://arxiv.org/pdf/1706.03762" -o arxiv_1706.03762.pdf

# NIST SP 800-53r5 — 492pp, ~6 MB, public domain (used for the PDF scale test)
curl -sL "https://nvlpubs.nist.gov/nistpubs/SpecialPublications/NIST.SP.800-53r5.pdf" \
     -o large_report_nist.pdf
```

### Optional bundle (not referenced by any runner)
`docs/mid_test_files.zip` was a convenience archive of markitdown's own MIT
test files; no test script reads it, so it is omitted from the repo. Pull the
individual files from
<https://github.com/microsoft/markitdown/tree/main/packages/markitdown/tests/test_files>
if you want them.

## Shared web HTML — re-fetch the originals
```bash
curl -sL "https://books.toscrape.com/"                         -o books_toscrape.html
curl -sL "https://quotes.toscrape.com/"                        -o quotes_toscrape.html
curl -sL "https://www.scrapethissite.com/pages/forms/"         -o scrapethissite_forms.html
curl -sL "https://en.wikipedia.org/wiki/Web_scraping"          -o wikipedia_web_scraping.html
```
(Live pages drift; the committed copies are the ones the recorded numbers were
measured against.)
